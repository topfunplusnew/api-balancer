
不错，现在我希望 apps/api/src/config/third-party-apis 这个目录用来存放第三方的API，每一个平台都有自己的文件夹，例如 apps/api/src/config/third-party-apis/creatomate 这个目录是存放creatomate这个平台的接口配置，目录下的config.js (原config.json 你需要进行修改) 会默认导出一个对象，这个对象就是关于这个第三方API的一些配置，如地址、版本、以及鉴权中间件。

在每一个第三方API的目录下 都有一个middlewares目录，这个目录用来存放对应第三方API的中间件，可以是鉴权中间件，也可以是数据转换中间件 等等，在config.js 默认导出的配置对象中，可以配置中间件，以及执行的顺序。还可以配置其对应的error处理器，当执行某个API报错的时候，执行对应的error处理器。还可以配置日志服务，这个日志服务你来设计，自由发挥。

现在，按照上面的理念，改一下apps/api/src/config/third-party-apis/creatomate，并且为这个 api 配置订单校验服务（我希望所有的第三方API都可以配置一个或者多个鉴权中间件，这些中间件按照顺序执行，并在配置中自定义顺序），订单校验服务需要写一个新的鉴权中间件，在这个鉴权中间件使用请求第三方webhook的方式，第三方webhook的配置如下（这个配置只是告诉你基本的信息，而并不是config.js导出的内容结构）：
{
	"url": "http://42.193.124.39:3030/api/sign",
	"method": "POST",
	"body": {
		"access_key": "{{parameters.access_key}}",
		"secret_key": "{{parameters.secret_key}}",
		"order_token": "{{parameters.order_token}}"
	},
	"response": {
		"data": {
			"access_key": "{{body.access_key}}",
			"secret_key": "{{body.secret_key}}"
		},
		"error": {
			"message": "[{{statusCode}}] {{body.error}}"
		}
	}
}
通过请求这个webhook可以获取校验结果，然后传递给下一个中间件，直到所有鉴权中间件处理完毕，返回最终结果。
